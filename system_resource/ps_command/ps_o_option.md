在 Linux 系统中，`ps` 命令的 `-o` 选项是一个非常灵活的工具。它允许你精准控制输出内容，只显示你关心的进程数据。这个功能在编写自动化脚本或排查特定性能问题时尤其有用，能帮你从大量信息中快速筛选出真正有价值的部分。

### 💡 如何使用 -o 选项

`-o` 的基本用法是 `-o field1,field2,field3,...`。你可以直接在 `-o` 后面用逗号分隔出一个字段列表，也可以使用多个 `-o` 选项。例如：
```bash
ps -eo pid,comm,pmem
```
这条命令会显示所有进程的 PID（进程ID）、COMMAND（命令名）和 %MEM（内存使用百分比）。其中 `-e` (`-A`) 选项表示显示所有进程。

你还可以用 `=` 符号来重命名列标题，让输出更清晰：
```bash
ps -eo pid=ProcessID,pmem=Mem%
```
这条命令会将 PID 列的标题改为 "ProcessID"，将 pmem 列的标题改为 "Mem%"。

### 📊 -o 支持的参数详解

`ps -o` 支持近百种不同的输出字段，下表整理了其中最常用的参数，按功能分成了几个大类：

| 参数 (Specifier) | 标题 (Header) | 描述 (Description) |
| :--- | :--- | :--- |
| **进程标识** |
| `pid` | PID | 进程的唯一标识符 |
| `ppid` | PPID | 父进程的标识符 |
| `tid` | TID | 线程的唯一标识符 |
| `pgrp` | PGRP | 进程所属组的ID |
| `session` | SESS | 会话ID |
| **基础与命令信息** |
| `user` / `uid` | USER / UID | 启动该进程的用户名（或其数字ID） |
| `group` / `gid` | GROUP / GID | 进程所属的用户组名（或其数字ID） |
| `comm` | COMMAND | 可执行文件的文件名（不含参数） |
| `cmd` / `args` | CMD / COMMAND | 完整的命令行（包含所有参数） |
| **CPU 资源相关** |
| `pcpu` / `%cpu` | %CPU | 该进程占用的CPU使用率百分比 |
| `cputime` / `time` | TIME | 进程自启动以来累计占用的CPU时间 |
| `etime` | ELAPSED | 进程从启动到当前时刻，经过的总时间 |
| `nice` / `ni` | NI | 进程的“nice”值，影响其调度优先级 |
| `pri` | PRI | 进程的内核调度优先级 |
| `psr` | PSR | 最近分配给此进程的CPU核心编号 |
| **内存资源相关** |
| `pmem` / `%mem` | %MEM | 该进程占用的物理内存百分比 |
| `vsize` / `vsz` | VSZ | 进程使用的全部虚拟内存大小（单位：KB） |
| `rss` | RSS | 进程当前驻留在物理内存中的大小（单位：KB） |
| `size` | SZ | 进程的代码+数据+栈大小的粗略估计 |
| `majflt` | MAJFLT | 进程发生的“缺页错误”（Major Fault）次数，通常意味着磁盘I/O |
| `minflt` | MINFLT | 进程发生的“次缺页错误”（Minor Fault）次数 |
| **时间与状态信息** |
| `lstart` | STARTED | 进程的启动时间（完整格式，含日期） |
| `start` | STARTED | 进程的启动时间（较短的格式） |
| `stat` / `state` | STAT / S | 进程的当前状态代码（如R=运行、S=睡眠、Z=僵尸等） |
| `tty` | TT | 进程所关联的终端设备 |
| `wchan` | WCHAN | 进程当前处于睡眠状态时，正在等待的内核函数名称 |

**💡 几点小提示：**
*   **格式规范**：参数名（Specifier）是你在 `-o` 命令中实际输入的内容，而标题（Header）是命令输出表格中的列名。
*   **参数变体**：部分参数有别名或变体，例如 `%cpu` 和 `pcpu` 都代表CPU使用率。
*   **查看完整列表**：如果你需要查看当前系统 `ps` 命令支持的所有字段，可以直接在终端执行 `ps L` 命令，它会在屏幕上列出所有可用的格式符。

### 🌰 实战示例

下面通过几个例子来演示 `-o` 的实际用法。

**1. 查看高CPU占用率进程**
```bash
ps -eo pid,user,pcpu,pmem,args --sort=-pcpu | head -n 10
```
这个组合命令 `-o` 指定了要查看的字段，`--sort=-pcpu` 按 CPU 使用率降序排序，`head -n 10` 则只输出前 10 行，能让你一眼看出是哪些进程在消耗CPU。

**2. 查看高内存占用进程**
```bash
ps -eo pid,user,pmem,rss,args --sort=-rss | head -n 10
```
这里 `-o` 指定了内存相关的内存列，并通过 `--sort=-rss` 按物理内存占用 (`rss`) 降序排列，同样用 `head` 筛选后，即可快速定位“内存大户”。

**3. 查看特定进程的详细信息**
```bash
ps -p 1234 -o pid,user,ppid,pcpu,pmem,etime,args
```
这个命令通过 `-p 1234` 锁定PID为 `1234` 的进程，然后用 `-o` 为其定制了一份专属的“体检报告”，涵盖了CPU、内存、运行时长等信息。

### 💎 总结

`ps -o` 赋予了你在 Linux 系统中查看进程信息的终极灵活性。通过组合使用不同的字段和强大的排序、筛选命令，你可以轻松构建出真正符合自己需求的信息视图，大幅提升工作效率。
