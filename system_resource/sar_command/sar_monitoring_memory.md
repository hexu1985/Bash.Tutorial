**核心结论**：使用 `sar -r` 命令是监控Linux系统内存最直接的方式，它可以清晰地展示物理内存和交换分区的使用情况，是定位内存瓶颈的重要工具。

---

### 一、内存监控核心命令

| 目标 | 命令 | 说明 |
| :--- | :--- | :--- |
| **查看内存使用情况** | `sar -r [interval] [count]` | 显示物理内存和交换分区的统计信息 |
| **查看内存页面统计** | `sar -R [interval] [count]` | 显示内存页面的统计信息 |
| **查看交换分区统计** | `sar -W [interval] [count]` | 显示页面交换统计数据 |
| **查看交换空间使用率** | `sar -S [interval] [count]` | 显示交换空间的使用率 |
| **保存监控数据** | `sar -r -o [filename] [interval] [count]` | 将结果**以二进制格式**存入文件，需要用 `-f` 读取 |
| **从文件读取数据** | `sar -r -f [filename]` | 从二进制文件中读取并显示内存报告 |
| **查看历史数据** | `sar -r -f /var/log/sa/saDD` | `DD`为当月日期，例如`sa15`代表当月15号的数据 |

---

### 二、典型输出示例与字段详解

以下命令每1秒采样一次，共采样3次，显示内存使用情况。

```bash
sar -r 1 3
```

**输出示例**：

```text
Linux 2.6.18-194.el5PAE (dev-db)     03/26/2011     _i686_     (8 CPU)

01:27:32 PM kbmemfree kbmemused  %memused kbbuffers kbcached  kbcommit   %commit
01:27:33 PM    123456   1543222     86.25     45678    345678   1250000     58.20
01:27:34 PM    123000   1543678     86.27     45700    345700   1251000     58.25
01:27:35 PM    122500   1544178     86.30     45720    345720   1252000     58.30
Average:       122985   1543693     86.27     45699    345699   1251000     58.25
```

**字段解释**：

| 字段 | 含义 | 健康参考值 |
| :--- | :--- | :--- |
| `kbmemfree` | 空闲物理内存（KB），**不包含** buffer 和 cache | 尽可能大 |
| `kbmemused` | 已用物理内存（KB），**包含** buffer 和 cache | - |
| `%memused` | 已用内存占总内存（不含swap）的百分比 | 持续 > 95% 需关注 |
| `kbbuffers` | 内核缓冲区占用的内存（KB） | 正常波动 |
| `kbcached` | 页面缓存占用的内存（KB） | 正常波动 |
| `kbcommit` | 系统承诺分配的内存总量（KB），含swap | 不超过物理内存+swap |
| `%commit` | 承诺内存占总内存（含swap）的百分比 | 可作为应用实际内存需求参考 |

> **关键解读**：`kbmemused`包含了buffer和cache，因此`%memused`很高是正常的。要计算应用实际使用的物理内存，可以用 `kbmemused` 减去 `kbbuffers` 和 `kbcached`。

### 三、内存页面统计（`sar -R`）

```bash
sar -R 1 3
```

```text
01:30:00 PM  frmpg/s  bufpg/s  campg/s
01:30:01 PM      0.00     0.00     0.00
01:30:02 PM      0.00     0.00     0.00
```

| 字段 | 含义 | 健康参考值 |
| :--- | :--- | :--- |
| `frmpg/s` | 每秒系统释放的内存页数；负值表示系统分配内存页 | 正常波动 |
| `bufpg/s` | 每秒系统为buffer分配的页数；负值表示系统在回收buffer空间 | 正常波动 |
| `campg/s` | 每秒系统为cached分配的页数；负值表示系统在回收cached空间 | 正常波动 |

### 四、保存与读取监控数据

#### 1. 保存数据

```bash
# 每1秒采样一次，共采样3次，将内存数据以二进制形式存入文件 'mem_info'
sar -r -o mem_info 1 3
```

该命令生成的是**二进制文件**，无法用 `cat` 正常查看，必须通过 `sar -f` 解析。

#### 2. 从文件读取数据

```bash
# 从二进制文件 'mem_info' 中读取并显示内存报告
sar -r -f mem_info
```

还可以与 `-s`（起始时间）、`-e`（结束时间）组合使用，提取指定时间段的记录：

```bash
# 读取文件，只显示 01:27:00 到 01:29:00 之间的数据
sar -r -f mem_info -s 01:27:00 -e 01:29:00
```

### 五、快速诊断脚本示例

以下脚本将监控数据保存到文件，并生成当日简要报告：

```bash
#!/bin/bash
# 保存实时数据（采样间隔60秒，共60次）
sar -r -o /tmp/mem_sar_$(date +%Y%m%d).bin 60 60

# 从文件读取并生成报告
echo "=== 内存使用摘要 ==="
sar -r -f /tmp/mem_sar_$(date +%Y%m%d).bin | tail -5
```

### 七、典型问题排查示例

**问题**：系统响应变慢，怀疑内存不足。

```bash
# 1. 查看内存使用趋势
sar -r

# 2. 若 %memused 持续>95%，进一步查看应用实际内存需求
sar -r -s 14:00:00 -e 15:00:00

# 3. 检查是否存在频繁swap
sar -W 1 5
```

### 总结

`sar -r` 命令是Linux内存监控的核心工具，它通过 `kbmemfree`、`kbmemused`、`%memused`、`kbcommit` 等指标，全面反映物理内存和交换分区的使用状况。**保存和读取数据是 `sar` 的核心价值之一**，其标准用法是通过 `-o file` 保存二进制数据，再用 `-f file` 进行事后读取分析。在生产环境中，建议结合 `sysstat` 后台数据收集服务（默认每10分钟采集一次），积累历史数据以便回溯定位问题。
